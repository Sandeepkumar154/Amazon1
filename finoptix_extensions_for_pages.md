## 3.3 Hardware And Software Requirements

### 3.3.1 Hardware Requirements
* **Processor (Server):** 8-Core Intel Xeon or AMD EPYC (Required for parallel inference)
* **GPU (Model Training/Inference):** NVIDIA RTX 3090 or A10G Tensor Core GPU (24GB VRAM required for Mamba sequence processing)
* **RAM (Server):** 32 GB DDR4 / DDR5
* **Storage (Server):** 500 GB NVMe SSD (High IOPS required for tick-level database writes)
* **Processor (Client):** Intel Core i3 or equivalent
* **RAM (Client):** 4 GB Minimum
* **Client Device:** Desktop, Laptop, or Tablet with minimum 1080p resolution for Optix Terminal rendering.

### 3.3.2 Software Requirements
* **Operating System (Server):** Ubuntu 22.04 LTS (Linux)
* **Operating System (Client):** Windows 10/11, macOS, Linux, or ChromeOS
* **Frontend Technologies:** Next.js 14, React.js, Tailwind CSS, Shadcn-UI, Lightweight Charts
* **Backend Technologies:** Python 3.10+, FastAPI, Uvicorn, Pydantic, WebSockets
* **Machine Learning Frameworks:** PyTorch, HuggingFace Transformers, Mamba-SSM CUDA Toolkit
* **Database Management:** PostgreSQL (via Supabase), Prisma ORM
* **Deployment & DevOps:** Docker, GitHub Actions, Vercel (Frontend), AWS EC2 (Backend)

### 3.4.2 Sequence Diagrams
Sequence diagrams capture the temporal flow of messages between objects.
* **WebSocket Market Data Flow:** The Client initiates a connection `ws://api.finoptix/stream`. The Server accepts and binds a broadcast channel. Continuously, the Backend Ticker Loop pulls data from the NSE/Yahoo bridge, identifies modified elements, and pushes `JSON` payloads specifically to subscribed Client IDs.
* **Prediction Generation Flow:** A CRON job triggers at 16:00 IST (Post-market). The Data Ingestor pulls the daily EOD OHLCV. The Feature Engineer script maps the 28 features. The tensor arrays are passed into the Mamba Model object. Predictions are saved via SQL Insert to the Predictions Database. The Frontend then queries `/api/predictions/latest` on the next day.

## 4.5 Detailed API Endpoint Implementations

To facilitate the decoupling of the frontend UI from the heavy AI backend, extensive API endpoints were architected. Below defines the structural contract for these interfaces:

### 4.5.1 Market Data Endpoints
* **`GET /api/v1/market/quote/{ticker}`**
  * **Description:** Retrieves the localized, cached snapshot of current market pricing.
  * **Response:** `{ "symbol": "INFY", "price": 1450.20, "change": -12.5, "pChange": -0.85, "timestamp": "..." }`
* **`GET /api/v1/market/history/{ticker}?timeframe=1D`**
  * **Description:** Fetches historical OHLCV data structures formatted explicitly for charting libraries.
  * **Response:** Array of objects containing UNIX timestamps and Open/High/Low/Close data.

### 4.5.2 Artificial Intelligence Endpoints
* **`GET /api/v1/ai/prediction/{ticker}`**
  * **Description:** Fetches the comprehensive Mamba-SSM analytical output for the current day.
  * **Response:** `{ "direction_raw": 0.85, "label": "STRONG_BUY", "trustability_score": 88, "features": { ... } }`
* **`POST /api/v1/ai/chat/query`**
  * **Description:** Accepts user text input, aggregates backend JSON state context, and interfaces with the LLM API.
  * **Payload:** `{ "user_id": "...", "chat_history": [...], "current_query": "Explain MACD." }`

### 4.5.3 Simulator Execution Endpoints
* **`POST /api/v1/simulator/trade`**
  * **Description:** Executes virtual order matching against live market prices. Highly validated to prevent negative balance manipulation.
  * **Payload:** `{ "ticker": "TCS", "action": "BUY", "quantity": 15 }`
  * **Internal Logic Array:** Validates user existence -> Queries live WS cache for actual TCS price -> Computes total cost -> Verifies virtual leverage -> Deducts balance -> Updates ledger.

## 4.6 Feature Engineering Algorithms (Pseudocode)

To explicitly demonstrate the methodology, the following pseudocode block represents the internal mechanism used to map standard Stock data into Mamba Features:

```text
ALGORITHM: Generate_Model_Features(StockDataArray)
Input: StockDataArray containing {Date, Open, High, Low, Close, Volume}
Output: Tensor matrix spanning 28 dimensions

BEGIN
    Matrix M = Empty Matrix [Length(StockDataArray), 28]
    
    // 1. Calculate Simple Returns
    For i from 1 to Length(StockDataArray):
        M[i]['Return'] = (Close[i] - Close[i-1]) / Close[i-1]
        M[i]['Log_Return'] = Log(Close[i] / Close[i-1])
        
    // 2. Exponential Moving Averages (EMA)
    M['EMA_9'] = Calculate_EMA(Close, 9)
    M['EMA_21'] = Calculate_EMA(Close, 21)
    M['EMA_200'] = Calculate_EMA(Close, 200)
    
    // 3. Volatility (Bollinger Bands)
    Rolling_StdDev = StandardDeviation(Close, 20)
    SMA_20 = Calculate_SMA(Close, 20)
    M['BB_Upper'] = SMA_20 + (Rolling_StdDev * 2)
    M['BB_Lower'] = SMA_20 - (Rolling_StdDev * 2)
    M['BB_Width'] = (M['BB_Upper'] - M['BB_Lower']) / SMA_20
    
    // 4. Momentum Oscillators (RSI)
    Average_Gain = RollingMean(Positive_Returns, 14)
    Average_Loss = Absolute(RollingMean(Negative_Returns, 14))
    RS = Average_Gain / Average_Loss
    M['RSI_14'] = 100 - (100 / (1 + RS))
    
    // 5. Volume Profile Adjustments
    M['VWAP'] = Cumulative(Close * Volume) / Cumulative(Volume)
    
    // 6. Normalization
    M_Scaled = RobustScaler(M)
    M_Standardized = Z_Score_Standardize(M_Scaled)
    
    RETURN M_Standardized
END
```
